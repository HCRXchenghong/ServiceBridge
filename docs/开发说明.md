# 开发说明

## 当前开工点

第一轮先完成项目骨架和真实业务闭环。

优先级：

1. 后端健康检查、登录、会话、消息、AI、关键词、备注、营业时间。
2. 用户端 Web 静态入口和 WebSocket 联调。
3. 客服端 uni-app 页面拆分与 API/WS 接入。
4. 管理端 uni-app 页面拆分与 API 接入。
5. WebSocket 协议文档。
6. PostgreSQL 表结构、仓库实现和 Redis 跨节点事件总线。
7. 核心业务边界测试和短压验证。

## UI 规则

三端 UI 必须按根目录三个示例实现。

- 用户端 Web：`用户端前端UI示例.html`
- 客服端 uni-app：`客服端前端UI示例.html`
- 管理端 uni-app：`管理端前端UI示例.html`

不要改成另一套视觉语言。

## 本地验证

后端：

```bash
cd backend
go test ./...
go run ./cmd/server
```

用户端：

```bash
cd apps/user-web
python3 -m http.server 5173
```

开发依赖：

```bash
POSTGRES_PORT=15432 REDIS_PORT=16379 docker compose -f deployments/docker-compose.dev.yml up -d postgres redis
```

PostgreSQL + Redis 后端：

```bash
cd backend
STORE_DRIVER=postgres \
DATABASE_URL='postgres://customer_service:customer_service_dev@localhost:15432/customer_service?sslmode=disable' \
REDIS_ADDR='localhost:16379' \
DATA_ENCRYPTION_KEY='local-dev-secret' \
go run ./cmd/server
```
