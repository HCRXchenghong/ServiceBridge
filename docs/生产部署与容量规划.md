# 生产部署与容量规划

目标：不考虑带宽瓶颈时，支撑 5000 个 WebSocket 并发连接，并能通过横向扩容继续提升容量。

## 推荐拓扑

- Nginx / 云负载均衡：负责 TLS、静态用户端 Web、HTTP API、WebSocket 升级。
- Nginx 示例已为用户端静态站配置基础安全头、CSP、静态资源长缓存和 WebSocket 反代头。
- Go Backend：无状态多实例，至少 3 个实例起步。
- PostgreSQL：保存账号、会话、消息、配置、关键词、营业时间。
- Redis：跨节点 WebSocket 事件广播；后续可继续承载在线状态 TTL、短锁和计数器。
- 上传存储：支持本地/共享卷和 S3/MinIO 兼容对象存储；大规模生产建议使用对象存储加 CDN。
- 推送服务：通过 `PUSH_WEBHOOK_URL` 对接 uni-push 云函数、APNs 或 Android 厂商推送适配服务。

## 单实例容量建议

- 单实例先按 2000-2500 条 WebSocket 长连接估算。
- 5000 并发建议至少 3 个后端实例：两台承载流量，一台冗余/滚动发布。
- 心跳间隔 30 秒，读超时 70 秒，反代 `proxy_read_timeout` 不低于 75 秒。
- WebSocket 写缓冲满时丢弃实时推送，但消息历史已经落库，客户端重连后通过历史接口补齐。

## Linux 参数

```bash
ulimit -n 200000
sysctl -w net.core.somaxconn=65535
sysctl -w net.ipv4.ip_local_port_range="10000 65000"
sysctl -w net.ipv4.tcp_tw_reuse=1
```

## 数据一致性

- 消息必须先写入数据库，再广播到 WebSocket。
- 客户端每条消息带 `client_msg_id`，服务端用 `conversation_id + sender + client_msg_id` 幂等去重。
- 会话转接和关闭必须在一个事务内释放旧客服计数、更新会话负责人、追加系统消息。
- 原始 IP 使用独立字段保存，备注不能覆盖 IP。
- 管理员/客服改密、客服被重置密码或禁用账号时，服务端会撤销该账号既有登录 token，并通过 Redis Pub/Sub 让各节点断开该账号在线 WebSocket。
- 浏览器 WebSocket 连接会校验 `Origin`，只允许 `CORS_ALLOWED_ORIGINS` 中的域名；原生 App/无 Origin 客户端按 token 鉴权。

## Redis 键设计

- `agent:online:{agent_id}`：客服在线状态，短 TTL。
- `agent:load:{agent_id}`：当前接待计数。
- `conversation:lock:{conversation_id}`：转接、关闭、派单短锁。
- `ws:conn:{node_id}:{client_id}`：连接所在节点。
- `pubsub:events`：跨节点事件广播。

## 压测顺序

1. 单实例 100、500、1000、2500 连接阶梯压测。
2. 多实例经 Nginx 压测 5000 连接。
3. 压测期间滚动重启一个后端实例，确认客户端重连恢复。
4. 压测期间模拟 Redis 短暂不可用，确认已落库消息不丢。
5. 压测期间模拟 PostgreSQL 慢查询，确认接口有超时和错误返回。

- 已实现数据概览聚合接口和服务评价闭环，避免管理端概览拉全量会话。
- 已实现图片上传、本地 `/uploads/` 静态访问和 S3/MinIO 兼容对象存储适配；Compose 示例默认用共享卷，生产建议设置 `UPLOAD_DRIVER=s3` 并绑定 CDN。

- 已实现用户端静态资源本地化，Nginx 示例已增加 CSP/安全头和 `/assets/` 长缓存。
- 已实现压测工具 ACK 延迟统计：`sent / acked / received / failed / ack_latency_*`。
- 当前已本机验证：内存模式 5000 WebSocket 短压通过，5000/5000 建连、0 失败、ACK P95 149ms；PostgreSQL+Redis 模式已做 30 连接短压。
- 已提供生产 Dockerfile、Nginx worker 配置、生产 Compose 示例和阶梯压测脚本。

## 关键生产环境变量

- `STORE_DRIVER=postgres`
- `DATABASE_URL=postgres://...`
- `REDIS_ADDR=redis:6379`
- `DATA_ENCRYPTION_KEY=<至少 32 字符的随机密钥>`
- `ADMIN_BOOTSTRAP_PASSWORD=<至少 12 字符的非默认管理员密码>`
- `AGENT_BOOTSTRAP_PASSWORD=<至少 12 字符的非默认客服密码>`
- `CORS_ALLOWED_ORIGINS=https://user.example.com,https://admin.example.com`
- `TRUSTED_PROXY_CIDRS=<Nginx/负载均衡所在网段>`
- `RATE_LIMIT_ENABLED=true`
- `METRICS_BEARER_TOKEN=<监控访问 token>`
- `OPENAI_API_KEY=<生产 Key>`
- `UPLOAD_DRIVER=s3`
- `S3_BUCKET=<对象存储桶>`
- `S3_PUBLIC_BASE_URL=<CDN/对象公开域名>`
- `PUSH_WEBHOOK_URL=<uni-push/APNs/厂商推送适配网关>`

生产环境启动时会执行配置体检：如果继续使用默认密码、开放 `CORS=*`、未设置监控 token、未启用限流、未配置 PostgreSQL/Redis 或可信代理网段不安全，后端会拒绝启动。
